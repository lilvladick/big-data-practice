Лекция 3. Введение в машинное обучение. Задача классификации. KNN. Метрики качества. Матрица ошибок
При работе с данными выделяют две группы признаков: прогнозирующие (входные) признаки — могут быть от одного до n, обычно легко измеримы; целевой (выходной) признак — обычно один, трудно измерим или неизвестен заранее, требует предсказания. Алгоритм строит решающее правило, предсказывающее значение целевого признака по значениям входных признаков. Если целевой признак количественный — задача регрессии; если категориальный — задача классификации. Примеры: для химических соединений входные признаки — особенности молекулярной структуры, выходной — токсичность или лечебные свойства; для зерна в сельском хозяйстве входные — свойства семян, грунта и погоды, выходной — урожайность или содержание клейковины; для банковских клиентов входные — демографические факторы и доход, выходной — факт потенциальной неплатёжеспособности.
Задачи с категориальными целевыми признаками подразумевают разделение множества объектов на классы. Решающее правило строится на обучающей выборке, где значения целевых признаков известны, и тестируется на тестовой выборке с неизвестными значениями. Процесс обучения направлен на минимизацию разницы между предсказанными и наблюдаемыми значениями целевого признака на обучающей выборке в классе допустимых решающих правил. Структура процесса: обучающая выборка поступает в алгоритм обучения, который формирует модель; тестовая выборка подаётся на вход модели для получения предсказаний; предсказания сравниваются с фактическими значениями для оценки качества.
Алгоритм k-ближайших соседей (kNN) относит новый объект к классу, который наиболее распространён среди k ближайших соседей в пространстве признаков. Расстояние между объектами обычно вычисляется как евклидово расстояние. Выбор параметра k критичен: малое k приводит к переобучению и чувствительности к шуму, большое k — к недообучению и игнорированию локальных паттернов. Перед применением kNN требуется нормализация признаков, так как признаки с большим масштабом будут доминировать в расчёте расстояния.
Логистическая регрессия решает задачу бинарной классификации, предсказывая вероятность принадлежности объекта к положительному классу. В отличие от линейной регрессии, которая плохо подходит для классификации (предсказывает значения вне диапазона [0,1], чувствительна к выбросам, не моделирует вероятности), логистическая регрессия использует сигмоидную функцию (функцию логистического преобразования), которая отображает линейную комбинацию признаков в диапазон (0,1). Модель основана на распределении Бернулли и обучается методом максимального правдоподобия. Для многоклассовой классификации применяются стратегии one-vs-all (обучение отдельной модели для каждого класса против всех остальных), all-versus-all (обучение модели для каждой пары классов) или многоклассовая логистическая регрессия (многономиальная логистическая регрессия).
Оценка качества моделей регрессии использует метрики: MSE (среднеквадратичная ошибка), RMSE (корень из MSE), MAE (средняя абсолютная ошибка), R² (коэффициент детерминации). Для моделей классификации сначала строится матрица ошибок (confusion matrix), содержащая четыре категории: истинно положительные (TP), истинно отрицательные (TN), ложно положительные (FP), ложно отрицательные (FN). Accuracy (точность) = (TP + TN) / (TP + TN + FP + FN) — доля правильных предсказаний. Precision (точность положительного класса) = TP / (TP + FP) — доля действительно положительных среди предсказанных как положительные. Recall (полнота) = TP / (TP + FN) — доля найденных положительных объектов среди всех положительных. F-мера = 2 * (Precision * Recall) / (Precision + Recall) — гармоническое среднее precision и recall, полезна при дисбалансе классов.
Для моделей, выдающих вероятности, строятся кривые: precision-recall curve (зависимость precision от recall при варьировании порога классификации) и ROC-кривая (зависимость true positive rate от false positive rate при разных порогах). Площадь под ROC-кривой (ROC AUC) интерпретируется как вероятность того, что случайно выбранный положительный объект получит оценку выше случайно выбранного отрицательного объекта. ROC AUC может быть некорректной метрикой при сильном дисбалансе классов, в таких случаях предпочтительнее использовать precision-recall curve и её площадь (PR AUC).
Проблема переобучения возникает, когда модель слишком хорошо подстраивается под обучающую выборку, включая шум и выбросы, и теряет способность обобщать на новые данные. Признаки переобучения: высокая точность на обучающей выборке и низкая на тестовой. Решения: использование кросс-валидации (например, k-fold — разбиение данных на k частей, обучение на k-1 частях и тестирование на оставшейся с циклическим перебором всех вариантов); регуляризация; упрощение модели; увеличение объёма данных. Предобработка данных включает обработку пропусков (удаление строк с пропусками, заполнение средним/медианой/модой, предсказание пропущенных значений), обработку выбросов (удаление, замена на граничные значения, трансформация), преобразование категориальных переменных (label encoding для порядковых признаков, one-hot encoding или dummy-кодирование для номинальных признаков). Для категориальных признаков с большим числом уникальных значений применяются методы понижения размерности или агрегации редких категорий.
